\subsubsection*{template$<$typename T$>$\newline
struct Binary\+Search\+Tree$<$ T $>$::\+\+Node}
It\textquotesingle{}s the basic building block of \doxylink{class_binary_search_tree}{Binary\+Search\+Tree}. 

Nodes store informations in BST. They are internally connected, via pointers to {\ttfamily \doxylink{struct_binary_search_tree_1_1_node_a1813c649caf64c9da85dfd53f11ee23a}{left}}, {\ttfamily \doxylink{struct_binary_search_tree_1_1_node_a5912d47e9cc94fb179a621aee86673f2}{right}} and {\ttfamily \doxylink{struct_binary_search_tree_1_1_node_ac784446c1bf53fa2a282e3e1f078d645}{parent}}. Left child of the node is smaller than it\textquotesingle{}s parent and right child is greater than the given node.

This implementation represents Nodes as a struct, because I didn\textquotesingle{}t see the purpose in creating the implementation of additional functions inside \doxylink{struct_binary_search_tree_1_1_node}{Node} that would be present in class representation.
