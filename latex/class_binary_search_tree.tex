\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename T$>$\newline
class Binary\+Search\+Tree$<$ T $>$}
Dynamic simple binary search tree implementation of interface \doxylink{class_i_tree}{ITree}. 

BST\textquotesingle{}s are sorted binary tree\textquotesingle{}s, in which each node follows the rule that:\+
\begin{DoxyItemize}
\item left child, should be lesser than it\textquotesingle{}s parent;
\item right child, should be greater than it\textquotesingle{}s parent;
\end{DoxyItemize}

This fact allows BST for quick data access, removal and addition time. To be specific it operates at O(log n) logarithmic time.


\begin{DoxyTable}{2}{}{}{1}
\SetCell{bg=\tableheadbgcolor,font=\bfseries}Operation&\SetCell{bg=\tableheadbgcolor,font=\bfseries}Time complexity\\
Addition&O(log n)\\
Deletion&O(log n)\\
Search&O(log n)\\
\end{DoxyTable}


BST represents data by objects of class \doxylink{struct_binary_search_tree_1_1_node}{Node} that stores data type T. \doxylink{struct_binary_search_tree_1_1_node}{Node}\textquotesingle{}s are internally connected and to go deeper inside tree, we use \doxylink{class_i_comparator}{IComparator} object, to easily determine the position of desired \doxylink{struct_binary_search_tree_1_1_node}{Node}.

\begin{DoxyNote}{Note}
In BST order of inserting elements matters and it influence the performance of operations on it. It is happening, because BST may degenerate to the form of link list, if order of elements is inefficient.
\end{DoxyNote}
